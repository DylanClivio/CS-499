Once I got the project running it showed impressive code coverage and had excellent potential to be integrated into a real business. The problem was it completely lacked any capability to do so. My goal was to do a complete overhaul and make this project into something I could use to help manage my own calendar. I may not be a business, but perhaps I could use this project to help me organize my appointments and tasks!

Algorithms and Data Structure

I made updates to my AppointmentService.java, ContactService.java, and TaskService.java in order to improve the algorithms and data structure of my code.

First, we’ll look at AppointmentService.java. My goal was to incorporate bulk operations to make the program scalable. I created “addAppointments” and “deleteAppointments”. This allows for better manipulation of multiple appointments at once.

addAppointment returns a boolean. This should improve error handling because this boolean represents the success or failure of the operation. Lastly, I created a “getAllAppointments” method that will print out a full list of the appointments

Next, we can look at TaskService.java. I improved the “getTask” method to include returning an optional task. This should make null handling more safe.

I used a functional interface “TaskUpdater” to reduce code duplication. This should also make it easier to incorporate future improvements. Lastly, I updated the “deleteTask” method by simplifying its logic

I believe these updates were thoughtful and challenging to incorporate. These updates not only provide the program with more capabilities and make it more efficient for the user but also make the code cleaner with a better potential for future improvements.

A lot of these improvements came about as I was researching for this module and I hadn’t originally planned on them. I knew from the beginning I wanted to incorporate bulk operations.

Databases

I have implemented a MySQL database to replace the hashmaps present in the original project.

The hashmaps incorporated into the original project are not compatible with a real-life use scenario because they lose all their memory every time the program closes. Incorporating MySQL will preserve the information and also provide better scalability and allow a user to find patterns in data.

this mySQL database contributes significantly to this project and completely changes its usability, scalability, and efficiency.

I learned how to connect MySQL to my Java project and TomCat server. A challenge I faced was understanding how the server would interact with the database. After entering information, the database would give a 404 error, not because anything was broken but because I simply didn’t add a step for the program to return to the original webpage or give a success notification after uploading to the database.
